# AirSENS — Karib Aerospace

**Aviation Maintenance & Sustainment System** · CAMO + MRO + Engineering

A production-grade fleet airworthiness platform with a distinctive dark "cockpit" UI.
Built with Vite + React 18 + TypeScript. Ships with a deterministic seeded fleet so
every screen is alive on first open — no backend required to demo.

---

## Quick start

```bash
npm install
npm run dev
```

Then open **http://localhost:5173**

To drop into your existing folder, copy the contents over your
`C:\Users\rilon\Documents\airsens_karib` directory (or just run it from here).

```bash
npm run build     # type-check + production build into dist/
npm run preview   # preview the production build
```

> Requires Node 18+ (Node 20/22 recommended).

## Modules

**Overview**
- Command Deck — live KPIs, 30-day utilization, fleet status, WO pipeline, priority aircraft

**CAMO · Airworthiness**
- Aircraft Management — register + slide-in aircraft profile drawer
- Component Management — hierarchical assembly tree with life/overhaul tracking
- Fleet Planning — 30/60/90-day maintenance forecast (Gantt-style)
- Logbook & Production — flight logs + utilization/load trends
- Aircraft Maintenance Program — MPD tasks, revisions, compliance
- AD / SB Tracking — directives with fleet compliance status
- MEL — categorised (A/B/C/D) dispatch items with rectification clocks
- Reliability Analysis — dispatch reliability, defects by ATA, MTBUR, trends
- Configuration Management — As-Designed → As-Built → As-Maintained timeline

**MRO · Execution**
- Work Orders — drag-and-drop Kanban (backlog → planned → in-progress → QA → closed)
- Inventory Management — stock levels, reorder + shelf-life alerts
- Tools & Manuals — calibration tracking + document control
- Sales & Invoice — multi-currency quotes and billing

**Engineering**
- **Amplification Factor Engine** — the signature analytical tool. Live sliders
  (payload, cycles, sector length, load factor, environment) drive a fatigue
  amplification model with cost multiplier, interval-adjustment %, driver
  attribution and a 10-year maintenance cost projection.
- Structural Repair Evaluation (DTE/DTI) — damage-tolerance inspection-interval calculator
- Ageing Aircraft Audit — AMC 20-20 / ICAO age-related programs

## The Amplification Factor model

`src/lib/amplification.ts` is a transparent, defensible model:

- **Payload** stress scales super-linearly (`payload^2.4`, a Miner's-rule proxy).
- **Cycle density** — short sectors mean more pressurisation/landing cycles per
  flight hour, increasing ground-air-ground damage.
- **Load factor** contributes linearly.
- **Environment** (1 benign → 5 harsh coastal/heat/dust) multiplies corrosion demand.

Outputs are log-compressed so realistic operations land in a **1.0×–2.6×** envelope,
mapped to bands: optimal / elevated / high / severe — each with an engineering
recommendation and projected lifecycle cost curve.

## Architecture

```
src/
  components/   Layout (sidebar/topbar), shared UI primitives
  data/         types + deterministic seed generator (14 aircraft, WOs, ADs…)
  lib/          Amplification Factor engine
  modules/      one file per screen (or grouped table modules)
  styles/       global cockpit design system (CSS variables)
```

All data is generated by a seeded PRNG (`seed.ts`, seed `20260601`) so the demo is
stable across reloads. Swap `seed.ts` for live REST calls to go production —
the typed domain model in `types.ts` is your API contract.

## Design system

Dark aerospace instrument theme — deep navy surfaces, neon cyan + amber accents,
**Sora** display type + **JetBrains Mono** for technical data, radar-grid ambient
background, glass panels with instrument corner-ticks, staggered fade-in motion.
All tokens live as CSS variables in `src/styles/global.css`.
